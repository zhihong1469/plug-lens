# 二、核心机制：帧ID + 人脸结果 强绑定（解决你的最大痛点）
这是整个架构的灵魂，**简单到极致，绝对不会错位**，我用最通俗的方式讲清楚：

## 1. 基础：每一帧都有唯一身份证 → `帧ID`
- 采集服务每生产一帧YUYV，自动分配一个**自增数字ID**（1、2、3、4...）
- 帧ID跟随帧数据存在主FrameLink中，永不重复

## 2. AI服务：推理结果必须「带ID出门」
1. AI拉取主链路**最新帧**（比如帧ID=5）
2. AI内部转换RGB、推理人脸
3. AI输出结果：`{帧ID=5，人脸坐标x1/y1/x2/y2，置信度}`
4. 把这个**纯数据**存到**AI专用链路**（覆盖式写入，只存最新一条）

## 3. 推流服务：按ID取结果，绝对配对
1. 推流拉取主链路**最新帧**（比如帧ID=5）
2. 推流去AI专用链路，找`帧ID=5`的人脸结果
3. 找到就配对成功，找不到就说明这帧还没推理完/无结果
4. 推流：要么发纯YUYV，要么发RGB+坐标，**永不乱配**

## 4. 时间差完美解决
- AI慢、推流快没关系：推流只认**帧ID**，对不上就跳过AI结果，只推视频
- 不需要队列、不需要缓存多帧，完全适配你的无回放需求
