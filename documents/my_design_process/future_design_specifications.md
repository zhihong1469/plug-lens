# 嵌入式C语言 跨平台通用工程设计 + 代码编写规范
**适配范围**：裸机（无OS）、RTOS（FreeRTOS/RT-Thread）、嵌入式Linux
**核心遵循**：你的现有工程架构 + 跨平台资源受限场景刚需
**约束**：纯设计标准，不编写任何代码，所有规则可直接用于工程校验

---

## 核心设计总纲（不可违反的铁律）
这是整个规范的底层逻辑，覆盖所有平台、所有模块：
1. **分层解耦**：单向依赖，禁止跨层逆向调用，硬件/系统/业务三层完全隔离
2. **双抽象层**：device（硬件抽象）+ OSAL（系统抽象），业务层**零平台依赖**
3. **内存静态化**：禁用`malloc/free`，全编译期静态内存分配
4. **无全局变量**：模块静态上下文 + 句柄传递，线程安全、可重入
5. **模块化封装**：接口（.h）与实现（.c）分离，单一职责、可裁剪
6. **资源闭环**：谁申请谁释放，初始化/销毁成对实现，无任何泄漏
7. **全链路优雅退出**：跨平台统一退出机制，阻塞接口可唤醒
8. **状态机管控**：全局+模块双层FSM，状态迁移标准化
9. **双总线解耦**：事件总线（控制）+ 数据总线（大数据），禁止裸传数据
10. **工业稳定**：无死循环、无句柄泄漏、无僵尸线程、异常可容错

## 一、工程目录结构规范（固定标准化）
基于你**实际的工程目录**优化，严格区分**平台无关核心代码**和**平台适配代码**，跨平台移植时**仅修改适配层，核心代码零改动**。

### 固定目录层级与职责
```
project/
├── src/                # 【平台无关核心】业务+架构核心，全平台通用
│   ├── app/            # 应用入口层：生命周期、总资源管控
│   ├── bus/            # 双总线：事件总线+数据总线
│   ├── fsm/            # 状态机：全局FSM+模块FSM
│   ├── link/           # 数据链路：帧池、队列、数据转发
│   ├── service/        # 业务服务：采集、AI检测等逻辑
│   └── device/            # device抽象接口：纯硬件无关标准定义
├── common/             # 【平台无关通用】基础组件，全平台复用
│   ├── configs/        # 全局配置：宏、参数定义
│   ├── log/            # 日志组件：分级打印、跨平台兼容
│   ├── pool/           # 静态内存池：替代动态内存
│   ├── queue/          # 静态队列：跨平台同步
│   ├── osal/           # 【新增】OSAL抽象接口：系统调用封装
│   ├── utils/          # 通用工具：字符串、校验
│   └── plugin/         # 插件框架：模块化加载管理
├── plugins/            # 【插件化实现】可裁剪硬件/业务插件
│   ├── app_plugins/    # 应用插件：交互、命令解析
│   ├── device_plugins/    # device驱动：USB摄像头/MIPI等硬件实现
│   └── service_plugins/# 业务插件：AI检测、图像处理
├── bsp/                # 【平台适配层】唯一存放平台专属代码
│   ├── linux/          # Linux适配：系统调用、V4L2、pthread
│   ├── rtos/           # RTOS适配：任务、信号量、驱动
│   └── bare/           # 裸机适配：寄存器、裸机驱动
├── third_lib/          # 第三方库：跨平台编译适配
├── config/             # 编译配置：Makefile、裁剪宏
└── output/             # 编译输出
```

### 目录设计强制规则
1. **平台隔离**：平台专属代码**仅允许放在bsp目录**，核心层禁止出现Linux/RTOS/裸机代码
2. **插件化**：硬件驱动、业务功能全部插件化，支持编译期裁剪
3. **核心不变**：`src/`+`common/` 全平台通用，移植不修改
4. **禁止耦合**：核心层不直接依赖BSP，仅通过device/OSAL交互

## 二、分层架构规范（跨平台通用七层架构）
在你原有**六层分层**基础上，**新增OSAL操作系统抽象层**，形成**严格单向依赖**的跨平台标准分层，是架构铁律。

### 标准七层分层（上层 → 下层 单向调用）
1. **应用入口层**：主函数、全局生命周期、资源总收口
2. **应用交互层**：用户交互、命令解析、业务触发
3. **业务服务层**：核心逻辑、状态机、总线调度
4. **数据链路层**：数据缓冲、内存池、线程/任务调度
5. **硬件抽象层（device）**：统一硬件接口，无平台实现
6. **操作系统抽象层（OSAL）**：统一系统接口，隔离内核/RTOS/裸机
7. **板级支持层（BSP）**：平台专属实现（最底层）

### 分层依赖铁律
1. 仅允许上层调用下层，**禁止下层调用上层、禁止跨层调用**
2. 业务/总线/状态机 **不直接操作硬件/系统**，仅依赖抽象层
3. 所有平台差异被BSP层封闭，核心层无感知

## 三、核心组件设计规范
### 1. 硬件抽象层（device）规范
适配你的`video_device`插件架构，跨平台硬件兼容核心：
1. **接口与实现完全分离**
   - `.h`：纯标准接口、枚举、句柄定义，无任何平台代码
   - `.c`：硬件实现放入插件，支持无缝替换
2. **驱动插件化**：统一注册接口，支持多设备、多硬件类型
3. **静态上下文**：无动态内存，硬件句柄映射静态内存索引
4. **标准接口**：固定实现 `init/open/start/stop/close/deinit`

### 2. 操作系统抽象层（OSAL）规范
**跨平台核心**，封装你现有Linux系统调用（pthread/pipe/select）：
1. **统一接口**：封装线程、锁、队列、IO、终端、信号
2. **平台适配**：Linux/RTOS/裸机各自实现，上层无感知
3. **阻塞可唤醒**：所有等待接口支持退出事件唤醒
4. **信号安全**：仅封装极简操作，禁止非信号安全函数

### 3. 内存管理规范（全静态化）
彻底禁用动态内存，适配资源受限平台：
1. **唯一分配方式**：通用静态内存池，编译期分配
2. **模块上下文静态化**：无全局结构体，内部`static`上下文
3. **句柄传递**：对外暴露索引句柄，无悬空指针
4. **大数据闭环**：视频帧通过数据总线+引用计数自动回收

### 4. 模块化封装规范
1. **无全局变量**：禁止全局裸变量/结构体，模块私有静态上下文
2. **句柄驱动**：所有模块标准接口：`create/init/run/deinit/destroy`
3. **可重入**：支持多实例，无静态共享冲突
4. **实现隐藏**：内部函数/变量加`static`，仅暴露必要接口

### 5. 双总线 + 状态机规范（继承你的核心架构）
#### 双总线
1. 事件总线：控制指令、状态变更（小数据）
2. 数据总线：视频帧等大数据（静态内存、引用计数）
3. 禁止裸指针传递数据

#### 状态机
1. 全局FSM总控 + 模块FSM子管控
2. 状态迁移查表执行，禁止裸写状态
3. 硬件启停、资源申请仅在状态机回调执行

## 四、代码编写强制规范（工业级标准）
### 1. 命名规范
1. 宏：全大写+下划线（`MAX_FRAME_NUM`）
2. 枚举/结构体：大驼峰+类型后缀（`VideoHandle_T`）
3. 函数：小驼峰+模块前缀（`videoHalInit`）
4. 静态私有：前缀`_`（`_static_ctx`）

### 2. 变量规范
1. 无全局变量，仅模块内部静态变量
2. 多线程共享变量：`volatile` 修饰
3. 使用C99标准类型（`uint8_t`），无平台相关类型

### 3. 函数规范
1. 入参校验、统一错误码返回
2. 单一出口，方便资源回收
3. 函数≤80行，单一职责
4. `.h`接口必写注释（功能/入参/返回值）

### 4. 退出与异常规范
1. 统一退出源：全局退出标志 + OSAL事件通知
2. 所有阻塞接口（select/poll/等待）必须支持退出唤醒
3. 退出时序：停止业务 → 销毁线程 → 关闭硬件 → 释放内存
4. 信号处理：仅置退出标志，无复杂逻辑

### 5. 多线程规范
1. 线程隔离，通过OSAL同步原语交互
2. 无死循环空转，阻塞等待降低CPU占用
3. 创建者负责销毁线程，无僵尸线程

## 五、跨平台适配规范
1. **最小改动**：移植仅修改`bsp/`、`device_plugin/`、`osal`实现
2. **编译宏隔离**：平台判断宏仅在适配层使用
3. **接口一致**：device/OSAL接口全平台统一
4. **可裁剪**：无需模块通过宏禁用，不删代码

## 六、工程合规校验标准（落地检查项）
1. 核心层无任何平台专属代码
2. 无`malloc/free`、无全局变量
3. 所有模块：句柄 + 成对初始化/销毁接口
4. 阻塞接口均可被退出事件唤醒
5. 无跨层调用、无资源泄漏
6. 全链路优雅退出，资源100%回收
