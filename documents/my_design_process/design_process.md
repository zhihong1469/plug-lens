# 嵌入式Linux视觉采集终端 V3.0 架构改革 **落地执行路线图（严格顺序、按步执行、不跳步、不返工）**
## 整体改革总原则
1. **不推翻重来、不批量移动文件、不删现有成熟组件**；
2. 先**物理目录微调** → 再**架构规范固化** → 再**核心OOP改造** → 再**冗余收敛轻量化** → 最后**适配编译&业务&提交**；
3. 全程只做**改名、职责收敛、接口重构、冗余裁剪**，90% 存量代码直接复用；
4. 先定架构规则，再写代码，杜绝改到一半迷茫跑偏。

---

# 第一阶段：基础铺垫（零业务改动，5分钟搞定）
## 步骤1：目录语义改名（仅改2处，不动任何文件代码）
### 执行动作
1. `src/hal` → 重命名为 `src/device`
2. `plugins/hal_plugins` → 重命名为 `plugins/device_plugins`

### 配套同步
1. 所有代码里 `#include "hal/xxx.h"` 批量改成 `#include "device/xxx.h"`
2. 各级 Makefile 里 `hal` 路径同步改成 `device`

### 目的
- 彻底去掉裸机 `HAL` 歧义，适配 Linux 应用层**设备抽象**定位；
- 物理目录先对齐新架构语义，为后续 OOP 设备基类铺路。

### 禁止做
不删文件夹、不移文件、不改任何业务逻辑代码。

---

## 步骤2：固化 V3.0 五层逻辑架构开发规范
直接把下面规则定为后续所有开发**强制铁律**，写代码前先对齐：
1. 逻辑五层：业务应用层 → 组件服务抽象层 → Linux系统适配层 → 通用工具层 → Linux内核层；
2. 物理只维护四层：**业务层、组件层、通用层、系统适配内嵌组件层**；
3. **所有硬件设备差异（USB/MIPI/网络摄像头）统一用 设备基类+OOP子类 实现**，不再搞独立重型HAL分层；
4. 双总线、状态机保留核心能力，**砍掉过度设计、无效预留、复杂嵌套**；
5. 系统适配层不单独建文件夹，**全部内嵌在 src/device 内部**，只做V4L2/mmap/ioctl 薄胶水封装，不重复造轮子；
6. 禁止模块内部自建裸线程、裸全局变量通信，统一走封装线程+总线/队列。

> 这一步是**架构灵魂定调**，后面所有编码都按这个约束来。

---

# 第二阶段：核心架构重构（整个改革最关键一步）
## 步骤3：重构 `src/device` 为 C-OOP 设备基类架构
### 执行动作
1. 在 `src/device/inc/` 定义**摄像头基类 + OPS操作表**（遵循你纯C OOP规范：基类首成员、const OPS表、函数指针接口）；
2. 提取所有摄像头**通用行为**：init / deinit / start / stop / get_frame / ctrl 统一抽象到基类；
3. 把原有 `plugins/device_plugins/video_usb` 改造为**USB摄像头子类**：
   - 内嵌基类作为第一个成员；
   - 用 `container_of` 做上下转型；
   - 子类 static 实现私有硬件逻辑。

### 核心价值
- 用 **OOP父子类天然替代旧HAL层**，保留硬件隔离能力，但去掉冗余分层；
- 正式确立：**组件服务抽象层是唯一允许写C-OOP的层级**。

### 禁止做
不改动 common、不改动 bus/fsm/link 现有代码，只做设备层抽象改造。

---

## 步骤4：落地 Linux 系统适配层（内嵌式薄封装）
### 执行动作
在 `src/device` 内部，对 V4L2 / open / ioctl / mmap / poll 做**极简薄封装**：
- 只封装繁琐、重复的系统调用；
- 不做二次多层包装、不造冗余结构体；
- 仅给摄像头子类提供底层基础接口。

### 关键定位
- 不新建文件夹、不独立分层；
- 系统适配层**内嵌在组件层的device内部**，完成逻辑五层里的第三层职责。

---

# 第三阶段：冗余层级收敛 & 轻量化裁剪
## 步骤5：收敛组件层各目录职责（只定规则，少改代码）
把原有 `src/service / src/link / src/bus / src/fsm` **逻辑上统一归入组件服务抽象层**：
1. `service`：业务服务入口，只依赖设备基类接口；
2. `link`：帧数据流转、内存池队列调度，不再做复杂路由预留；
3. `bus`：保留事件/数据总线**核心解耦、零拷贝**能力，裁剪无用高级预留特性；
4. `fsm`：删掉强制全局+模块双层状态机嵌套，改为**按需轻量化状态机**，不用的业务不硬套状态机。

### 执行原则
- 物理文件夹全部保留；
- 只**收敛职责、砍掉无效预留、限制过度嵌套**，不删成熟代码。

## 步骤6：裁剪无用冗余预留代码
### 清理项
1. `src/link/audio_link` 空预留目录直接归档/删除；
2. 各模块里为未来预留的无用接口、空函数、冗余分支全部清理；
3. 去掉 fsm、bus 里为车规级复杂场景设计的闲置逻辑。
