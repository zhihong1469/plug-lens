# 不依赖OpenCV前提下，接下来要做的全部工作（按顺序整理）
## 模块1：V4L2摄像头采集适配
1. 枚举你的USB摄像头**支持的分辨率**，确认是否原生支持 320×240 YUYV；
2. 若支持：直接把配置宏改成 320×240，固定YUYV格式；
3. 若不支持：保留640×480采集，后续加纯C缩放；
4. 锁定采集帧率为 **15~20FPS**，和AI推理速率对齐；
5. 完善帧缓冲区管理，固定BUF数量，避免内存碎片。

## 模块2：纯C手写图像预处理（替代OpenCV）
1. 编写 **YUYV422 → RGB888** 纯C转换函数；
2. 如需降级：编写 **640×480 → 320×240** 纯C图像缩放（最近邻插值即可，够用、计算量小）；
3. 编写 RGB 像素**归一化、排布调整**，适配RFB-320模型的输入张量格式；
4. 全部封装成独立工具函数，和解码、业务逻辑解耦。

## 模块3：线程架构重构（适配单核+主线程休眠）
1. 架构定型：
   - 主线程：初始化 + 休眠控权 + 信号处理（退出/暂停）；
   - 子线程1：V4L2 摄像头循环采集；
   - 子线程2：AI模型循环推理；
2. 实现**环形帧队列**，做线程间帧数据传递、限流丢帧；
3. 配置线程调度策略，绑定CPU核心、降低主线程调度优先级；
4. 实现信号捕获（SIGINT），优雅唤醒主线程、回收子线程、释放资源。

## 模块4：RFB-320 INT8 模型工程适配
1. 下载对应的 INT8 量化模型文件，放到工程指定路径；
2. 移植推理框架（MNN/TFLite 二选一，推荐MNN更轻量）；
3. 编写模型初始化、推理接口，对接预处理后的RGB数据；
4. 输出人脸检测框坐标、置信度，预留后续扩展接口。

## 模块5：性能适配与单核优化
1. 强制推理框架**设置单线程推理**，关闭内部多线程；
2. 关闭摄像头自动增益、白平衡等多余算法，减少CPU开销；
3. 调整线程休眠间隔，避免空轮询占用CPU；
4. 测试实际帧率、CPU占用、内存占用，做微调。

## 模块6：工程化与编译部署
1. 整理项目目录架构，贴合你六层分层规范；
2. 编写Makefile/CMake，配置交叉编译链；
3. 剥离多余依赖，保证**零OpenCV、零冗余库**；
4. 开发板上部署模型、可执行文件，上电自启测试。

## 模块7：预留后续扩展
1. 预留绘图接口（以后想画框再补极简绘图函数，不用OpenCV也能手写）；
2. 预留人脸识别、上报TCP数据的扩展入口；
3. 整理代码注释，方便毕设答辩、秋招面试讲解。
